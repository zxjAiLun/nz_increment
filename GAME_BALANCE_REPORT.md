## 二、问题诊断

### 严重问题

#### 1. 防御增长彻底失衡

- **怪物防御**公式: `baseValue * 3` → 难度 500 时达 **32,509**
- **玩家穿透**仅从幸运值获得: `floor(luck * 0.1)` → 满幸运 1000 仅提供 **100 点穿透**
- **有效防御** = 怪物防御 - 玩家穿透 → 难度 500 下 **99.4% 减伤**
- 玩家攻击 1,010，仅能造成 ~6 点伤害，而怪物 HP = **108 万**

#### 2. 生命偷取形同虚设

- 当前绑定于幸运值: `luck * 0.008%`，最高 8%/次
- 高难度下普攻伤害个位数，回血量可忽略（~0.4 HP/次）
- 无独立生命偷取装备词条或技能来源（技能 `life_steal` 描述为"恢复30%"但未在伤害计算链中体现）

#### 3. 暴击率成长无感知

- 基础 5%，每难度 +0.01%，上限 30%（代码 cap）
- 难度 2500 才达到 30% 上限，普通游戏进程完全无感
- 幸运值可提供额外 `luck * 0.08%` 暴击，但同样需要大量投入

### 中等问题

#### 4. 装备等级 vs 怪物难度脱节

- 装备等级 = 击杀怪物的难度值（公式: `level: difficultyValue`）
- 低难度装备在高难度怪物面前属性不值一提
- 无装备等级锁定/惩罚机制，玩家无法携带"怀旧装"挑战

#### 5. 速度属性战斗存在感弱

- 虽然战斗规则写明"速度比≥2先手双动"，但实际战斗（gameStore.ts）未完全实现此机制
- 速度主要影响行动槽填充速度，但优势不够显著

## 三、优化建议

### 3.1 防御/穿透重定价

**怪物防御增长率调整**: 将 `baseValue * 3` 改为 `baseValue * 1.5`

| 难度值 | 旧怪物防御 | 新怪物防御 | 降幅 |
|--------|-----------|-----------|------|
| 100 | 121 | 60 | -50% |
| 300 | 1,986 | 993 | -50% |
| 500 | 32,509 | 16,254 | -50% |

**玩家穿透独立成长**: 每 50 难度 +5 基础穿透

```typescript
// 新增：独立穿透成长线
const basePenetrationFromDifficulty = Math.floor(difficultyValue / 50) * 5
// 加上幸运穿透
totalPenetration = basePenetrationFromDifficulty + calculateLuckPenetrationBonus(luck)
```

预期效果：难度 500 时玩家拥有 ~55 穿透（10 幸运 + 50 难度成长），vs 怪物防御 16,254 → 仍有 ~97% 减伤，但配合真实/虚空伤害可玩。

### 3.2 生命偷取重设计

```typescript
// 独立 lifesteal 属性（替代 luck × 0.008%）
interface LifestealSource {
  base: number      // 基础 0%
  equipment: number // 装备加成 0.5%~5%
  skill: number     // 技能加成 1%~15%
  total: number    // 求和后 cap 50%
}

// 示例装备词条
{ type: 'lifesteal', value: 3, isPercent: true }  // 吸血套2件效果
{ type: 'lifesteal', value: 10, isPercent: true } // 吸血套4件效果
```

### 3.3 暴击率重定价

- 移除 30% 上限（代码中 `Math.min(critRate, 30)`），提升至 75%
- 基础暴击率调整为每难度 +0.02%（旧）+ 幸运值 `luck * 0.1%`

### 3.4 抽奖成本重定价

```typescript
// 旧：单抽 = 100 × 1.005^difficulty
// 新：单抽 = 50 × 1.002^difficulty
const lotteryCost = Math.floor(50 * Math.pow(1.002, difficultyValue))
```

| 难度值 | 旧单抽成本 | 新单抽成本 | 降幅 |
|--------|-----------|-----------|------|
| 100 | 164 | 61 | -63% |
| 300 | 446 | 82 | -82% |
| 500 | 1,210 | 111 | -91% |
