# 数值平衡优化规格文档

## Why

根据项目分析，当前游戏存在严重的数值平衡问题：

1. **怪物防御增长过快** - 难度1000时防御达13,608，普通攻击几乎无效
2. **抽奖成本增长过快** - 难度1000时需29,017次击杀才能单抽一次
3. **穿透属性获取不足** - 玩家无法有效对抗后期防御
4. **真实伤害依赖过度** - 后期战斗完全依赖真实伤害/虚空伤害

这些问题导致玩家体验在后期急剧下降，需要系统性修复。

## What Changes

### 1. 怪物属性平衡调整

#### 1.1 防御力系数调整
- **当前**: `defense = baseValue * 6`
- **修改为**: `defense = baseValue * 3`
- **影响**: 难度1000时防御从13,608降至6,804

#### 1.2 怪物暴击率上限调整
- **当前**: `critRate上限 = 50%`
- **修改为**: `critRate上限 = 30%`
- **影响**: 降低玩家被暴击秒杀的风险

#### 1.3 BOSS属性倍率调整
- **当前**: `bossAttack = normalAttack * 2`, `bossDefense = normalDefense * 2`
- **修改为**: `bossAttack = normalAttack * 1.5`, `bossDefense = normalDefense * 1.2`
- **影响**: BOSS不再过于变态，保留挑战性

### 2. 抽奖系统平衡调整

#### 2.1 抽奖成本增长率优化
- **当前**: `lotteryCost = 100 * (1.05 ^ difficulty)`
- **修改为**: `lotteryCost = 100 * (1.02 ^ difficulty)`
- **影响**: 难度1000时成本从131,579,000降至约550,000

#### 2.2 抽奖奖励优化
- **增加**: 高级属性（四维）的出现概率
- **增加**: 金币奖励的基础数额
- **调整**: 装备/属性/金币的掉落比例

### 3. 穿透属性增强

#### 3.1 装备穿透词条权重提升
- **当前**: 穿透词条生成概率较低
- **修改为**: 进阶及以上装备必定有穿透词条
- **影响**: 玩家可通过装备获得足够穿透

#### 3.2 幸运值对穿透的加成
- **新增**: `penetrationBonus = luck * 0.1`
- **影响**: 高幸运玩家获得额外穿透

### 4. 战斗系统改进

#### 4.1 速度优势实现
- **新增**: 当玩家速度 > 怪物速度 * 1.5时，获得先手优势
- **新增**: 当玩家速度 > 怪物速度 * 2时，额外获得10%伤害加成
- **影响**: 速度属性更有价值

#### 4.2 生命偷取效果修复
- **修复**: 技能"生命汲取"现在正确触发生命偷取
- **公式**: `lifesteal = damageDealt * (lifeStealRate / 100)`
- **影响**: 战斗续航能力提升

### 5. 装备系统优化

#### 5.1 装备等级缩放调整
- **当前**: `levelScale = 1.15 ^ (difficulty / 50)`
- **修改为**: `levelScale = 1.12 ^ (difficulty / 50)`
- **影响**: 装备成长更平滑

#### 5.2 低稀有度装备价值提升
- **新增**: common装备现在有10%概率额外生成一条词条
- **新增**: good装备现在有20%概率生成2条词条
- **影响**: 低稀有度装备仍有使用价值

## Impact

### 受影响的文件

| 文件 | 修改内容 |
|------|---------|
| `src/utils/monsterGenerator.ts` | 怪物属性公式 |
| `src/utils/equipmentGenerator.ts` | 装备生成公式 |
| `src/stores/playerStore.ts` | 抽奖系统、幸运加成 |
| `src/utils/calc.ts` | 生命偷取、速度优势 |
| `src/gameStore.ts` | 战斗循环逻辑 |
| `README.md` | 更新数值文档 |

### 风险评估

| 风险 | 影响 | 缓解措施 |
|------|------|---------|
| 玩家战力过强 | 游戏过于简单 | 动态调整怪物属性 |
| 抽奖过于频繁 | 经济膨胀 | 设置每日抽奖上限 |
| 装备获取过快 | 失去挑战动力 | 调整装备掉落率 |

## 验收标准

### 必须通过 (Must Pass)

- [ ] 难度1000时，怪物防御不超过10,000
- [ ] 难度1000时，单抽成本不超过1,000,000金币
- [ ] 玩家攻击能在合理时间内击杀当前难度怪物
- [ ] 速度优势（>2倍）时伤害加成生效
- [ ] 生命偷取技能正确触发

### 建议通过 (Should Pass)

- [ ] 装备穿透词条出现率 > 30%
- [ ] 幸运500时获得50+穿透加成
- [ ] 抽奖平均收益 > 成本的80%

## 回滚方案

如果修改后出现以下情况，将触发回滚：

1. **游戏过于简单**: 如果平均击杀时间 < 0.5秒
2. **经济崩溃**: 如果金币通货膨胀 > 1000倍
3. **玩家流失**: 如果难度0的新玩家无法正常游戏

回滚方法: 恢复 `monsterGenerator.ts`, `equipmentGenerator.ts`, `playerStore.ts` 到修改前版本
