## 一、核心数值设计

### 1. 防御/穿透平衡

**问题诊断**（当前 nz_increment）：
- 怪物防御增长率 1.15^(d/10)，远超玩家穿透成长
- 后期普攻几乎无效

**参考设计**（来自 Idle Mineralocracy、Adventure Capitalist）：
- 防御/穿透使用 **对数成长** 而非指数
- Kongregate  Idle Game Math 论文指出：指数增长（n^x, n>1）最终会超过任何多项式增长（x^k），无论 k 多大或 n 多小
- 公式建议：防御 = base × (1 + difficulty × 0.01)
- 每 50 难度穿透 +5 的线性成长
- 成本增长率建议使用 1.07（AdVenture Capitalist 标准值），而非当前过高的倍率

**具体改进**：
```
当前问题：monster_def = base × 1.15^(d/10)
建议修改：monster_def = base × (1 + d × 0.02)
穿透成长：flat_penetration += d × 0.1 (每难度 +0.1 穿透)
```

### 2. 暴击率/暴伤曲线

**当前问题**：
- 基础 5%，需 difficulty 4500 才接近 50% cap
- 暴击成长感知不明显

**参考设计**（来自 Realm of the Mad God、原神 Genshin Impact）：
- 原神社区验证的最优暴击率:暴击伤害 = 1:2 比例（例：50%暴击率 + 100%暴击伤害）
- 暴击伤害应设置 **软上限** 和 **硬上限**，防止数值无限膨胀
- DPS 公式参考：DPS = 基础伤害 × 暴击率 × (1.5+暴击效果) × 攻速 + 基础伤害 × 不暴击率 × 攻速
- 前 500 难度暴击率快速成长（每 10 难度 +1%），之后缓慢成长（每 50 难度 +1%）

**具体改进**：
```
暴击率成长：
  d < 500:  每 10 难度 +1% (基础 5% + d/10 × 1%)
  d >= 500: 每 50 难度 +1% (封顶 80%)
暴击伤害：150% + d × 0.05 (封顶 300%)
```

### 3. 生命偷取机制

**当前问题**：
- 绑定幸运值，实际几乎为 0
- 缺乏独立恢复手段

**参考设计**（来自 Diablo、Hades、Wynncraft）：
- 生命偷取应为 **独立属性**，通过技能/装备获取
- Wynncraft 的设计：life steal scales per-hit health with attack speed，listed amount ÷ attack speed
- 建议公式：lifesteal = skill_bonus + equipment_bonus
- 上限建议 15-20%（Diablo 常见值）
- 生命偷取应与攻击力挂钩，而非与幸运值绑定

**具体改进**：
```
新增独立属性：lifesteal_rate (基础 0%)
获取途径：
  - 技能树解锁：+2%/技能
  - 装备套装：2件套 +3%
  - 紫色武器词缀：+5%
上限：15%
```

### 4. 抽奖成本

**当前问题**：
- 100 × 1.005^difficulty，后期无法支撑
- 玩家与怪物防御力失衡导致金币收入下降

**参考设计**（来自 Cookie Clicker、Adventure Capitalist）：
- 使用 **对数成本** 或更低增长率（1.001-1.002）
- AdVenture Capitalist 成本倍率 1.07 为行业标准
- 引入 **免费抽奖** 机制（每 N 分钟一次）
- 离线挂机收益保护

**具体改进**：
```
当前：cost = 100 × 1.005^d
建议：cost = 50 × 1.003^d 或 cost = 50 × 1.07^(d/10)
免费单抽：每 30 分钟一次（每日 48 次免费机会）
```

### 5. 经验值/升级曲线

**参考设计**（来自 Kongregate Idle Game Math）：
- 升级成本应使用 **多项式增长** 而非指数
- 反向平衡法：定义 "预期时间" 和 "标准值"，然后推导所需数值
- 每个 generator 的 cost_growth 建议 1.07-1.15

## 二、战斗系统

### 6. 速度先手 / ATB 系统

**参考设计**（来自 Final Fantasy ATB、Cogmind）：
- ATB (Active Time Battle) 由 Hiroyuki Ito 在 FF4 中发明，2010 年专利过期
- Cogmind 的时间系统：每回合 = 100 时间单位，动作消耗可变
- 速度计算公式（FF7 参考）：unit_speed = battle_speed × dexterity / party Dexterity
- 速度越快，时间单位消耗越少，可行动频率越高

**nz_increment 建议实现**：
```
速度条机制：
  每 100 速度 = 1 回合时间单位
  速度 200 的角色 = 速度 100 角色的 2 倍行动频率
  速度比 >= 2: 先手偏移 = (fast - slow) × 0.3
  速度比 >= 3: 双动 + 先手
  速度比 < 1: 敌方先手
```

### 7. 伤害弹出（Damage Popup）

**参考设计**（来自 Hades、Diablo 4）：
- 不同伤害类型不同颜色：白字(普攻)、黄字(暴击)、紫字(真伤)、绿字(治疗)
- 暴击时字体放大 + 闪烁 + 震动
- 溢出数字用 K/M 缩写（1,234,567 → 1.23M）
- Hades 暴击时屏幕轻微震动 + 粒子特效

**具体实现建议**：
```javascript
damagePopup = {
  normal: { color: '#FFFFFF', scale: 1.0 },

  heal: { color: '#2ECC71', float: 'up' },
  lifesteal: { color: '#E74C3C', float: 'up' }
}
```

## 三、装备与技能

### 8. 套装效果

**参考设计**（来自 World of Warcraft、Diablo、Perfect World）：
- Perfect World: 多件套装提供 **递进式奖励**，穿部分也有效果
- 知乎经验：装备系统遵循"目标-策略-价值"三层设计
- 分层设计原则：将装备数据拆分为基础属性、套装效果、品质系统等模块

**nz_increment 套装设计**：
```
套装系统（3件/5件）：
  战士之力套装：
    2件：攻击力 +10%，防御 +5%
    3件：生命偷取 +3%
    5件：必杀一击（攻击时 5% 概率造成 300% 伤害）
  
  游侠敏捷套装：
    2件：攻速 +8%，闪避 +3%
    3件：暴击率 +5%
    5件：连击（攻击时 10% 概率触发双倍攻击）
  
  法师智慧套装：
    2件：技能伤害 +15%，蓝耗 -10%
    3件：冷却缩减 +10%
    5件：奥术飞弹（普通攻击 5% 概率释放连锁闪电）
```

### 9. 被动技能设计

**参考设计**（来自 Path of Exile、Slay the Spire）：
- Slay the Spire 开发者 Anthony Giovannetti 的数据驱动设计：
  - 关键指标 1：玩家选择卡牌的频率（太低 = 牌组中不存在）
  - 关键指标 2：卡牌出现在获胜牌组中的频率（太高 = 失衡）
- PoE 的被动技能树提供 **选择而非加成**，强调构建路线
- 0 费卡通常比 1 费卡弱，但配合特定机制可产生协同

**nz_increment 被动技能建议**：
```
被动技能应提供条件性增益：
  条件触发型：
    "攻击速度 > 2.0 时，暴击率 +15%"
    "生命值 < 30% 时，受到伤害 -20%"
    "连击数 > 5 时，伤害 +25%"
  
  阈值型：
    "每拥有 100 点防御，反击伤害 +5%"
    "每 10% 暴击率，暴击伤害 +10%"
```

### 10. 技能连招 / 协同效应

**参考设计**（来自 Hades、Bastion）：
- Hades: 技能间有明确的 **协同效应**（Hammer 强化 + 武器形态变化）
- 技能A 标记敌人，技能B 对标记敌人伤害 +50%
- 鼓励玩家研究技能组合，而非无脑堆数值

**nz_increment 技能协同设计**：
```
技能A「穿刺」：对敌人施加"穿刺标记"（持续 3 回合）
技能B「引爆」：对带标记敌人造成额外 80% 伤害
技能C「贯穿」：标记时间延长至 5 回合，额外 +20% 伤害
套装协同：战士之力 5 件使「穿刺」有 20% 概率眩晕敌人 1 回合
```

## 四、内容扩展

### 11. 练功房难度设定

**参考设计**（来自各类 MMORPG 刷怪点）：
- 最佳收益难度 = 玩家战力 × 0.8-1.2
- 难度过低：收益减少 50%
- 难度过高：死亡率上升，经验减半

**具体实现**：
```
推荐难度范围：player_power × (0.8 ~ 1.2)
超出范围惩罚：
  低于 0.5×: 金币 -50%，经验 -30%
  高于 1.5×: 受到伤害 +100%，死亡概率 +30%
```

### 12. 每日/每周挑战

**参考设计**（来自各类手游）：
- 每日随机 3 个挑战
- 完成奖励：钻石/金币/装备
- 挑战类型：击杀数、不受伤害通关、时间限制

**每日挑战系统**：
```
每日挑战池（每日随机 3 个）：
  1. 速战速决：在 60 秒内击败当前难度怪物
  2. 无伤通关：完成 10 回合战斗不失血
  3. 连击达人：单次战斗达成 20 连击
  4. 财源滚滚：击杀怪物获得双倍金币
  5. 暴击盛宴：使用暴击击杀 50 个怪物

奖励：
  完成 1 个：50 金币
  完成 2 个：100 金币 + 1 抽奖券
  完成 3 个：200 金币 + 2 抽奖券 + 随机绿装
```

### 13. 月卡/战令设计

**参考设计**（来自原神、崩坏：星穹铁道）：
- 月卡：每日登录奖励 + 离线收益加成
- 战令：任务解锁奖励，分免费/付费档

**nz_increment 商业化设计**：

月卡（30 元/月）：

战令（68 元/赛季）：
  基础奖励：完成任务获得积分，解锁奖励档
  免费档：基础装备强化材料

## 六、Slay the Spire 技能平衡方法论

**核心原则**（来自 Mega Crit Games 官方 GDC 分享）：
- **数据驱动**：跟踪两个关键指标
  1. 玩家选择卡牌的频率（pick rate）— 低于某值说明牌不存在
  2. 卡牌出现在获胜牌组的频率（win rate）— 高于某值说明失衡
- **不要追求所有卡牌平等**：游戏可以在总体平衡的前提下存在强弱卡
- **PVE 平衡比 PVP 简单**：单游戏可接受部分强力组合存在
- **迭代测试**：调整数值 → 测试 → 再调整

**对 nz_increment 的启示**：
- 在游戏内埋点，记录玩家使用频率
- 定期分析数据，识别过强/过弱的技能
- 每 2-4 周进行一次平衡性更新
- 设置 "体验服" 频道，新内容先在那测试

## 八、关键数值公式汇总

```javascript
// 防御成长（当前问题）
const current = {
  monsterDef: (d) => baseDef * Math.pow(1.15, d / 10),
};

// 推荐修改
const recommended = {
  monsterDef: (d) => baseDef * (1 + d * 0.02),
  playerPenetration: (d) => basePenetration + d * 0.1,
};

// 暴击成长
const critRate = (d) => {
  if (d < 500) return 0.05 + d / 1000; // 每10难度+1%
  return 0.50 + (d - 500) / 5000; // 每50难度+1%，封顶80%
};
const critDamage = (d) => Math.min(1.5 + d * 0.05, 3.0); // 封顶300%

// 抽奖成本
const gachaCost = (d) => 50 * Math.pow(1.003, d);

// 生命偷取
const lifesteal = {
  base: 0,
  skillBonus: 0.02, // per skill
  setBonus: 0.03, // 2-piece
  weaponAffix: 0.05,
  cap: 0.15, // 15%上限
};

// 速度条
const speedRatio = (playerSpeed, enemySpeed) => playerSpeed / enemySpeed;
const initiative = (ratio) => {
  if (ratio >= 3) return 'double_action';
  if (ratio >= 2) return (ratio - 2) * 0.3; // 先手偏移
  if (ratio < 1) return 'enemy_first';
  return 'normal';
};
```
